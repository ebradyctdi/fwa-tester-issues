Work Instruction: Pallet Audit Tool

FWA Operations — CTDI Coatesville

1. Purpose

This work instruction describes how to use the Pallet Audit Tool to verify the contents of a pallet before shipment. The audit confirms that all expected IMEIs are physically present on the pallet and records the audit result.

2. Before You Begin

  • Navigate to the Pallet Audit Tool page (Audit Tools → Pallet Audit Tool)
  • Ensure your barcode scanner is connected and functioning
  • Have the pallet manifest (list of expected IMEIs) available — these will be scanned during setup
  • Confirm the connection status indicator shows green (Connected)

3. Audit Process

Step 1: Scan Pallet ID

  1. Place your cursor in the Pallet ID field
  2. Scan the pallet's barcode label (11 characters, starts with "PLB")
  3. The system will validate the format and advance to the next step

Step 2: Scan Part Number

  1. Scan the Part Number barcode on the pallet
  2. Valid part numbers:
    • WNC-CR200A — Titan 3
    • ASK-NCM1100 — Titan 4
  3. The model type will display next to the part number

Step 3: Scan IMEI Barcodes (Pallet Manifest)

  1. Scan each IMEI barcode from the pallet manifest (the expected units)
  2. Each IMEI must be 15 characters — invalid lengths will show an error
  3. Duplicate IMEIs are automatically rejected
  4. The scan count updates as you add each IMEI
  5. When all expected IMEIs are scanned, click "Begin Audit"
Note: You can scan any number of IMEIs for the manifest. There is no hard limit.

Step 4: Perform the Audit (Scan Physical Units)

  1. The system enters Audit Mode — you will now scan the actual units on the pallet
  2. Select your Scan Mode using the toggle buttons:
    • Quality Check & IMEI Record (green, default) — IMEI is recorded AND marked as quality inspected. Unit turns green in the table.
    • Just IMEI Record (light blue) — IMEI is recorded only. Unit turns light blue in the table.
  3. Scan each unit's IMEI barcode or QR code (both are supported)
  4. You can switch between scan modes at any time during the audit
  5. For each scan:
    • Match found (Quality): Unit marked green ✅
    • Match found (IMEI only): Unit marked light blue ☑️
    • Not on manifest: Alert appears — you can "Record as Issue" or "Ignore - Do Not Record Unit"
    • Already scanned: Duplicate warning (not counted again)
  6. Monitor the KPI cards at the top:
    • IMEIs on Pallet — total expected
    • Quality Checked During Audit — units quality inspected
    • Matched During Audit — total units verified (quality + IMEI only)
    • Remaining — still to scan
    • Issues Logged — problems recorded
Tip: Use "Quality Check & IMEI Record" for the units you physically inspect. Use "Just IMEI Record" for units you verify by barcode/QR only without opening the box.

Step 5: Report Issues (If Needed)

  1. If an unexpected IMEI is scanned, you have two options:
    • "Record as Issue" — logs the unexpected IMEI as an issue
    • "Ignore - Do Not Record Unit" — dismisses without recording
  2. To report a standalone issue, click the red "Report Issue" button
  3. Enter the IMEI and describe the issue in the popup
  4. Issues are logged to the Pallet Audit Issues sheet

Step 6: Complete the Audit

  1. When all units have been scanned, click "Complete Audit"
  2. A summary popup shows audit metrics (match rate, issues, etc.)
  3. Enter the Auditor Name (required)
  4. Add any Notes (optional)
  5. Select the audit result:
    • OK To Ship (green) — Pallet passes audit (disabled if open issues exist)
    • Hold for Action (red) — Pallet fails audit
  6. Click the result button to save the audit
Important: You cannot select "OK To Ship" if there are any open issues associated with the pallet. All issues must be resolved first.

Step 7: Print Audit Label

  1. After saving, the Print Label button appears
  2. A 4×6 label preview is shown with the audit result, pallet details, and a barcode
  3. Click "Print Label" to send to the Zebra ZD421 printer
  4. Affix the label to the pallet

4. Starting a New Audit

Click "Reset / New Audit" at the top of the page to clear all data and begin a fresh audit for the next pallet.
